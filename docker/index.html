Issue tracker redirector

API:

  • GET /browse/:jira_key - Redirects to the corresponding GitHub issue, if found. Otherwise to Jira. (Recommended, other routes are kept for compatibility reason)
  • GET /:jira_key - Redirects to the corresponding GitHub issue, if found. Otherwise to Jira.
  • GET /issue/:jira_key_id - Redirects to the corresponding GitHub issue, if found. Otherwise to Jira prefixing it with JENKINS-.

Example of usage:

From a Jira link like issues.jenkins.io/browse/JENKINS-1, replace its subdomain by issue-redirect: issue-redirect.jenkins.io/browse/JENKINS-1

To add more Jira redirects send a pull request to jira_keys_to_github_ids.txt.
